Your Team will build a simple video game using Scratch. The link to Scratch is https://scratch.mit.edu. The link for the tutorial for building a game is https://www.youtube.com/watch?v=8RIJqEqZ_zU.

The submission for this should be a link to your online Scratch video game, in an MS Word file. If you create your game online, you will be using Scratch 3, which is the most versatile version.

An alternative is to download Scratch, which will give you Scratch 2. This provides a choice to create your game offline. In this case, you can submit your entire game as a .sb file.
📁 Deliverable Structure
DELIVERABLE STRUCTURE

CHOOSE ONE PATH — Online or Offline
│
├── PATH A — Online (Scratch 3 · Recommended)
│   ├── Go to scratch.mit.edu and create account     ← free, browser-based
│   ├── Build game using block-based coding           ← follow tutorial video
│   ├── Publish project to get a share link           ← click "Share" in Scratch
│   ├── Paste link into a Word document (.docx)       ← this is your submission file
│   └── Submit the .docx to Canvas                   ← file types: docx / doc / odt
│
└── PATH B — Offline (Scratch 2)
    ├── Download Scratch from scratch.mit.edu        ← gives you Scratch 2
    ├── Build game offline                            ← follow tutorial video
    ├── Save game as a .sb file                       ← the whole game is one file
    └── Submit the .sb file directly to Canvas       ← no Word doc needed

△  Game must be playable — not a blank or broken project.
△  Tutorial: youtube.com/watch?v=8RIJqEqZ_zU  — watch before building.
